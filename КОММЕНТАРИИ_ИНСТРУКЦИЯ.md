# Инструкция по добавлению системы комментариев

## Что было добавлено

### Backend (Python/FastAPI)

1. **Миграция БД** - `backend/add_comments_migration.py`
   - Создаёт таблицу `comments` с полями:
     - `id` - PRIMARY KEY
     - `user_id` - FOREIGN KEY к users.id
     - `user_login` - логин пользователя
     - `content` - текст комментария (до 2000 символов)
     - `target_type` - тип объекта ('general', 'test_result', 'consultation')
     - `target_id` - ID объекта для привязки
     - `parent_id` - ID родительского комментария (для древовидной структуры)
     - `created_at`, `updated_at` - временные метки
     - `is_deleted` - мягкое удаление
   - Создаёт индексы: `idx_comments_target`, `idx_comments_user`, `idx_comments_parent`

2. **Модель SQLAlchemy** - `backend/main.py`
   - Добавлен класс `Comment` с relationships для пользователя и ответов

3. **Pydantic схемы** - `backend/main.py`
   - `CommentCreate` - для создания комментария
   - `CommentUpdate` - для обновления комментария

4. **API эндпоинты** - `backend/main.py`
   - `POST /api/comments` - создать комментарий
   - `GET /api/comments?target_type=X&target_id=Y` - получить комментарии
   - `GET /api/comments/{id}` - получить конкретный комментарий
   - `PUT /api/comments/{id}` - обновить комментарий (только автор)
   - `DELETE /api/comments/{id}` - удалить комментарий (автор или админ)
   - `GET /api/admin/comments` - все комментарии (админ)
   - `DELETE /api/admin/comments/{id}` - административное удаление

### Frontend (React)

1. **API клиент** - `frontend/src/api/api.js`
   - `commentsAPI` - CRUD операции для комментариев
   - `adminAPI.getAllComments()` - получить все комментарии
   - `adminAPI.deleteCommentAdmin(id)` - удалить комментарий (админ)

2. **React компонент** - `frontend/src/components/Comments.jsx`
   - Отображение списка комментариев
   - Создание новых комментариев
   - Ответы на комментарии (древовидная структура)
   - Редактирование своих комментариев
   - Удаление своих комментариев
   - Интеграция с темой (light/dark)

3. **Интеграция в страницы**
   - `Dashboard.jsx` - комментарии на вкладке профиля
   - `TestResults.jsx` - комментарии к результатам теста

## Как запустить миграцию

### Шаг 1: Активируйте виртуальное окружение

```bash
cd backend
.\venv\Scripts\activate  # Windows
# или
source venv/bin/activate  # Linux/Mac
```

### Шаг 2: Запустите миграцию

```bash
python add_comments_migration.py
```

Ожидаемый вывод:
```
✅ Таблица comments успешно создана
📋 Структура:
  - id: PRIMARY KEY
  - user_id: INTEGER (FK -> users.id)
  ...
✅ Индексы созданы: idx_comments_target, idx_comments_user, idx_comments_parent
```

### Шаг 3: Перезапустите backend

```bash
python main.py
```

### Шаг 4: Проверьте работу

1. Откройте `http://localhost:3000` (frontend)
2. Войдите в аккаунт
3. Перейдите в личный кабинет (`/dashboard`)
4. Прокрутите вниз - там должен быть блок "Комментарии"


## Типы комментариев

| target_type | Описание | target_id |
|-------------|----------|-----------|
| `general` | Общие комментарии | `null` |
| `dashboard` | Комментарии в личном кабинете | `null` |
| `test_result` | Комментарии к результатам теста | `code` (PSY-...) |
| `consultation` | Комментарии к заявке | ID консультации |

## Безопасность

- Только авторизованные пользователи могут создавать комментарии
- Редактировать/удалять можно только свои комментарии
- Администратор может удалять любые комментарии
- Мягкое удаление (`is_deleted`) - данные сохраняются в БД
- XSS защита через `sanitize_string()`
- Максимальная длина комментария - 2000 символов

## Возможные проблемы и решения

### Ошибка "table comments already exists"
Таблица уже создана.Migration уже была запущена.

### Ошибка "no such table: comments"
Миграция не была запущена. Выполните Шаг 2.

### Комментарии не отображаются
1. Проверьте, что backend запущен
2. Откройте консоль браузера (F12) и проверьте ошибки
3. Убедитесь, что вы авторизованы

### 401 Unauthorized
Требуется авторизация. Войдите в аккаунт.

## Удаление функционала (если нужно)

```sql
DROP TABLE IF EXISTS comments;
```
